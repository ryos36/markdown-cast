# チュートリアル

README の「始め方」（字幕つき動画を作るところ）の続きです。
Azure TTS での音声合成、作業の効率的な進め方、各種調整について説明します。

---

## Azure TTS で音声を付ける

### Azure TTS とは

Microsoft Azure が提供する音声合成（Text to Speech）サービスです。
テキストを自然な音声に変換できます。日本語では複数の音声が用意されており、
声の高さや速さも調整できます。

markdown-cast では発話ノートを Azure TTS で音声にします。
利用するには Azure Portal で API キーを取得する必要があります。

### API キーの入手

Azure Portal から Speech のリソースを作ると API キーが取得できます。
無料枠があるので、まずは無料で試せます。
API キーは外部に漏れないように管理してください。

### API キーの設定

`intro/key.ninja` に API キー・リージョン・音声名を書きます。

```
key    = ここに API キーを貼る
region = japaneast           # Azure のリージョン
voice  = ja-JP-NanamiNeural  # 使用する音声
```

`key.ninja` は `.gitignore` に登録済みなので、git を使っている場合でも
誤ってコミットしてしまう事故は防がれています。

### 音声の種類・高低・速さ

`voice` で音声の種類を選べます。音声の一覧は Azure Portal の Voice Gallery で確認できます。

声の高さや速さは `tts_opt` で調整できます。

```
tts_opt = --pitch -10% --rate 5%
```

- `--pitch`: 声の高さ（`-10%` で低く、`+10%` で高く）
- `--rate`: 速さ（`-15%` でゆっくり、`+15%` で速く）
- `--style`: 話し方のスタイル（`cheerful`、`calm` など）

### 音声つき動画を作る

キーを設定したら、次のコマンドで音声つき動画を作ります。

```sh
ninja video-audio
```

`_build/intro-audio.mp4` に音声つき動画ができます。

## 音声合成を効率よく進めるために

全ページ分の音声を一度に作ると時間も費用もかかります。
2 つの機能を組み合わせると費用を抑えながら進められます。

### 1. `limit` でページ数を絞る

`key.ninja` の `limit` で音声を作るスライドのページ数を制限できます。

```
limit = 6   # 最初の 6 ページだけ
limit = 0   # 全ページ
```

### 2. `--dry-run` で音声合成をスキップする

`key.ninja` の `tts_opt` に `--dry-run` を設定すると、
Azure を呼ばずにパイプライン全体の流れを確認できます。

```
tts_opt = --dry-run
```

`--dry-run` では音声の長さが固定値（1 文節あたり約 200ms）になるため、
字幕と音声のタイミングはずれます。タイミングは気にせず、スライドの内容だけ確認します。

### 基本の進め方（4 ステップ）

**step 1: 最初の数ページで音声を確認する**

`limit = 6` にして最初の数ページだけ音声化します。
音声はタイミングや読み方など細かい点が気になりやすいので、
最初の数ページで確認してから全ページに進むのが無難です。

**step 2: スライドの内容を仕上げる**

スライドの内容の編集に集中します。
このあいだ音声合成を完全に止めたい場合は `tts_opt = --dry-run` にします。
あるいは `ninja video` で音声なしの動画だけ作って確認する方法もあります。

**step 3: 音声の最終確認**

`limit` を有効にしたまま、全体の音声を確認します。
読み方や間のとり方に問題がないか確認します。

**step 4: 全ページの音声を作る**

問題がなければ `key.ninja` の設定を戻します。

```
limit   = 0   # 全ページ
tts_opt =     # --dry-run を外す（空にする）
```

全ページ分の音声を作るので時間がかかります。費用もかかります。
無料枠の場合、途中で上限に達するかもしれません（無料枠は毎月リセットされます）。

## 音声と字幕の調整

### 調整できるもの

1. **文節の区切り** — 字幕の更新タイミングと音声の間
2. **句読点** — 字幕・音声に含めるかどうか
3. **読み** — 固有名詞などの読み方
4. **build.ninja のパラメータ** — 表示時間・余白・速度調整など

### 1. 文節の区切り

動画の中で字幕は文節ごとに更新されます。スライドの背景画像はそのままで、
字幕テキストだけが変わります。音声の中でも文節の間に短い間が入ります。

文節の区切りは形態素解析で自動的に決まりますが、意図と違う場合は 2 つの方法で調整できます。

**全角空白で区切り位置を指定する**

発話ノートに全角空白「　」を入れると、その位置で文節を区切れます。

```
micro:bit を　使って　みよう
```

**複合語を 1 語として扱う**

`micro:bit` のような語が複数に分割されてしまうときは、
`share/mecab-private.dict.ss` に登録すると 1 語として扱えます。

```lisp
("micro:bit" "micro:bit")
```

### 2. 句読点

句読点（。、）で文が自動的に区切られます。1 つの文が音声の 1 つの読み上げ単位になります。

`build.ninja` の `strip_opt` で字幕・音声に含めるかどうかを設定します。

```
strip_opt = --strip-punctuation   # 句読点を落とす（既定）
strip_opt =                       # 句読点を残す
```

句読点を残すと、Azure が読み上げ時にブレス（息継ぎ）を入れます。

### 3. 読み

`share/pronunciation.dict.ss` で固有名詞などの読み方を指定できます。

```lisp
("micro:bit" . "マイクロビット")
```

読みの変更は音声にだけ反映されます。字幕は元の表記のまま表示されます。

### 4. build.ninja のパラメータ

| パラメータ | 説明 | 既定値 |
|---|---|---|
| `capms` | 1 フレームの表示時間（ms） | 800 |
| `head_ms` | 各文の音声前の余白（ms） | 50 |
| `tail_ms` | 各文の音声後の余白（ms） | 10 |
| `threshold` | 速度調整の上限係数 | 1.3 |
| `max_chars` | 字幕 1 行の最大文字数 | 20 |
| `break_ms` | 文節間の間（ms） | 120 |

**capms**: 1 フレーム（字幕 1 枚）の表示時間と動画のフレームレート（`1000/capms` fps）を両方決めます。

**head_ms / tail_ms**: 各文の音声の前後に余白を設けます。

**threshold**: 音声が長すぎて速度を上げる場合、倍率がこの値を超えるとフレームを追加して
速めすぎを防ぎます。フレームが追加されると、その文の最後の字幕が繰り返し表示されます。

**max_chars**: 字幕 1 行の最大文字数。超えると次の文節から新しい字幕になります。

**break_ms**: 音声内の文節と文節の間に入れる間（ms）。
短くすると早口になり、長くするとゆったりした読み上げになります。
