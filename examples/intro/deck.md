## markdown-cast: どうつかうの？

このツールは 4 つのステップで動画ができます。

1. 作業ディレクトリと markdown-cast を用意する
2. `deck.md` にスライドと発話ノートを書く
3. ビルドする
4. できた動画を再生する

<!--

4 つのステップで動画ができます。

-->

---

## 作れる動画の種類

markdown-cast で作れる動画は 2 種類あります。

- **字幕つき動画** — Azure TTS なしで作れます
- **音声つき動画** — Azure TTS を使います

音声の合成には Microsoft Azure の Text to Speech（TTS）サービスを使います。
まず字幕つき動画で内容を確認し、
準備ができたら音声つき動画を作る流れをお勧めします。

<!--

## 初期設定1. 作業ディレクトリを用意する

スライドや音声ファイルを置くディレクトリを作り、
markdown-cast を git submodule として追加します。

```sh
mkdir my-slides && cd my-slides
git init
git submodule add https://github.com/ryos36/markdown-cast
```

ディレクトリ名（ここでは `my-slides`）は自由につけられます。
スライドの内容には影響しません。

# Azure TTS で音声を付ける

---

## Azure TTS とは？

Microsoft Azure が提供する音声合成（Text to Speech）サービスです。

テキストを自然な音声に変換できます。
日本語では複数の音声が用意されており、
声の高さや速さも調整できます。

markdown-cast では発話ノートを Azure TTS で音声にします。
利用するには Azure Portal で API キーを取得する必要があります。

<!--
Azure TTS は Microsoft Azure の音声合成サービスです。
テキストを自然な音声に変換できます。
markdown-cast では発話ノートを Azure TTS で音声にします。
利用するには API キーが必要です。
-->

---

## API キーの入手

Azure Portal から Speech のリソースを作ると API キーが取得できます。

無料枠があるので、まずは無料で試せます。

API キーは外部に漏れないように管理してください。

<!--
Azure Portal で Speech のリソースを作ると API キーが取得できます。
無料枠があるのでまずは無料で試せます。
API キーは外部に漏れないよう注意してください。
-->

---

## Tips: API キーの管理

`key.ninja` に API キーを書きます。

git を使っている場合、`key.ninja` は自動的に `.gitignore` に登録されています。
誤って git にコミットしてしまう事故を防ぐ仕組みです。

git を使っていない場合でも、`key.ninja` の取り扱いには注意してください。
API キーが漏れると、無断で音声合成が使われる可能性があります。

<!--
key.ninja に API キーを書きます。
key.ninja は gitignore 登録済みなので git にはコミットされません。
git を使っていない場合でも API キーの管理には注意してください。
-->

---

## API キーの設定

`my-first-slide/key.ninja` に API キー・リージョン・音声名を書きます。

```
key    = ここに API キーを貼る
region = japaneast           # Azure のリージョン
voice  = ja-JP-NanamiNeural  # 使用する音声
```

<!--
my-first-slide/key.ninja に API キーとリージョンと音声名を書きます。
-->

---

## 音声の種類・高低・速さ

`voice` で音声の種類を選べます。音声の一覧は Azure Portal の Voice Gallery で確認できます。

https://speech.microsoft.com/portal/voicegallery

声の高さや速さは `tts_opt` で調整できます。

```
tts_opt = --pitch -10% --rate 5%
```

- `--pitch`: 声の高さ（`-10%` で低く、`+10%` で高く）
- `--rate`: 速さ（`-15%` でゆっくり、`+15%` で速く）
- `--style`: 話し方のスタイル（`cheerful`、`calm` など）

<!--
voice で音声を選び、tts_opt で声の高さや速さ、話し方を調整できます。
-->

---

## はじめはページ数を絞って試す

全ページ分の音声を一度に作ると時間も費用もかかります。

`key.ninja` の `limit` で音声を作るスライドのページ数を制限できます。

```
limit = 6   # 最初の 6 ページだけ
limit = 0   # 全ページ
```

まず数ページだけ音声を作って確認することをお勧めします。

<!--
limit は音声を作るスライドのページ数を制限する設定です。
まず数ページだけ音声を作って確認することをお勧めします。
-->

# 音声合成をうまく進めるために

<!--
音声合成を効率よく進めるためのポイントを説明します。
-->

---

## 費用を抑えながら進める 2 つの方法

1. **`limit`** — 音声を作るページ数を制限する
2. **`--dry-run`** — 音声合成をスキップして処理の流れを確認する

この 2 つを組み合わせることで、費用を抑えながら作業を進められます。

<!--
費用を抑えながら進めるための 2 つの方法を紹介します。
limit でページ数を制限し、dry-run で音声合成をスキップして処理の流れを確認します。
-->

---

## step1: 最初のページで音声を確認する

`limit = 6` で最初の 6 ページだけ音声化します。

音声は細かい点が気になりやすいので、
最初の数ページで内容を確認してから全ページに進むのが無難です。

<!--
step 1 は limit を使って最初の数ページだけ音声化します。
音声はけっこう気になるので、まずここで確認しておくのが無難です。
-->

---

## step2: スライドの内容を仕上げる

スライドの内容の編集に集中します。

音声の最終調整はスライドが仕上がってから行います。

<!--
step 2 はスライドの内容の編集に集中します。
音声の調整はスライドが仕上がってから行います。
-->

---

`limit` を設定していても、条件によっては音声合成が実行されることがあります。
（なるべく実行しないようにはしています）

スライド編集中に音声合成を完全に止めたい場合は `--dry-run` を使います。

```
tts_opt = --dry-run
```

<!--
limit を設定していても条件によっては音声合成が実行されることがあります。
完全に止めたい場合は dry-run を使います。
-->

---

`--dry-run` では音声の長さが固定値（1 文節あたり約 200ms）になります。

そのため字幕と音声のタイミングがずれます。
タイミングは気にせず、スライドの内容だけ確認します。

あるいは `ninja video` で音声なしの動画だけ作って確認する方法もあります。

<!--
dry-run では音声の長さが固定値になりタイミングがずれます。
タイミングは気にせず、スライドの内容だけ確認します。
あるいは ninja video で音声なしの動画だけ作って確認することもできます。
-->

---

## step3: 音声の最終確認

`limit` を有効にしたまま全体の音声を確認します。

読み方や間のとり方に問題がないか確認します。

<!--
step 3 では limit を有効にしたまま全体の音声を確認します。
読み方や間のとり方を確認します。
-->

---

## step4: 全ページの音声を作る

問題がなければ `key.ninja` の設定を戻します。

```
limit   = 0   # 全ページ（制限を外す）
tts_opt =     # --dry-run を外す（空にする）
```

`limit` は `key.ninja` の変数、`tts_opt` も `key.ninja` の変数です。
どちらも同じファイルで管理しています。

全ページ分の音声を作るので時間がかかります。費用もかかります。
無料枠の場合、途中で上限に達するかもしれません（毎月リセットされます）。

<!--
問題がなければ key.ninja の limit を 0 に、tts_opt を空にして全ページの音声を作ります。
全ページ分の音声を作るので時間がかかり費用もかかります。
-->

# 音声と字幕の調整

<!--
ここから音声と字幕の各種調整について説明します。
-->

---

## 調整できるもの

1. **文節の区切り** — 字幕の更新タイミングと音声の間
2. **句読点** — 字幕・音声に含めるかどうか
3. **読み** — 固有名詞などの読み方
4. **build.ninja のパラメータ** — 表示時間・余白・速度調整など

<!--
調整できるものは 4 種類あります。
文節の区切り、句読点、読み、そして build.ninja のパラメータです。
-->

---

## 調整1: 文節の区切り

動画の中で字幕は文節ごとに更新されます。
スライドの背景画像はそのままで、字幕テキストだけが変わります。

音声の中でも文節の間に短い間が入ります。

文節の区切りは自動的に決まりますが、
意図と違う場合は調整できます。

<!--
動画では字幕が文節ごとに更新されます。
スライドの背景はそのままで字幕テキストだけが変わります。
音声にも文節の間に短い間が入ります。
区切りは自動で決まりますが、意図と違う場合は調整できます。
-->

---

文節の区切りが意図と違う場合、2 つの方法で調整できます。

1. **全角空白** — 発話ノートに「　」を入れる
2. **複合語辞書** — `mecab-private.dict.ss` に登録する

<!--
文節の区切りが意図と違う場合は、全角空白か複合語辞書で調整できます。
-->

---

## 全角空白で区切り位置を指定する

発話ノートに全角空白「　」を入れると、その位置で文節を区切れます。

```
（例）
micro:bit を　使って　みよう
         ^^   ^^^^ ここで区切れる
```

<!--
全角空白を入れるとその位置で文節を区切れます。
-->

---

## 複合語を 1 語として扱う

`micro:bit` のような語が複数の語に分割されるときは、
`mecab-private.dict.ss` に登録すると 1 語として扱えます。

```lisp
("micro:bit" "micro:bit")
```

<!--
複合語が分割される場合は mecab-private.dict.ss に登録します。
-->

---

## 調整2: 句読点

句読点（。、）で文が自動的に区切られます。
1 つの文が音声の 1 つの読み上げ単位になります。

`build.ninja` の `strip_opt` で字幕・音声に含めるかどうかを設定します。

```
strip_opt = --strip-punctuation   # 句読点を落とす（既定）
strip_opt =                       # 句読点を残す
```

句読点を残すと、Azure が読み上げ時にブレス（息継ぎ）を入れます。

<!--
句読点で文が区切られ、1 つの文が音声の 1 つの読み上げ単位になります。
strip_opt で句読点を字幕と音声に含めるかどうかを設定します。
句読点を残すと Azure がブレスを入れ、自然な読み上げになります。
-->

---

## 調整3: 読み

`pronunciation.dict.ss` で固有名詞などの読み方を指定できます。

```lisp
("micro:bit" . "マイクロビット")
```

読みの変更は音声にだけ反映されます。字幕は元の表記のまま表示されます。

<!--
pronunciation.dict.ss で固有名詞などの読み方を指定できます。
読みの変更は音声にだけ反映され、字幕は元の表記のまま表示されます。
-->

---

## 調整4: build.ninja のパラメータ

| パラメータ | 説明 | 既定値 |
|---|---|---|
| `capms` | 1 フレームの表示時間（ms） | 800 |
| `head_ms` | 各文の音声前の余白（ms） | 50 |
| `tail_ms` | 各文の音声後の余白（ms） | 10 |
| `threshold` | 速度調整の上限係数 | 1.3 |
| `max_chars` | 字幕 1 行の最大文字数 | 20 |
| `break_ms` | 文節間の間（ms） | 120 |

<!--
build.ninja のパラメータで動画の細かい挙動を調整できます。
-->

---

## capms: フレームの長さ

`capms` は 1 フレーム（字幕 1 枚）の表示時間です。

字幕の表示時間と動画のフレームレート（`1000/capms` fps）を両方決めます。

既定は 800ms（1 秒に 1.25 フレーム）。

<!--
capms は 1 フレームの表示時間です。
字幕の表示時間と動画のフレームレートを両方決めます。
-->

---

## head_ms / tail_ms: 各文の余白

各文の音声の前後に余白を設けます。

```
|←    head_ms    →|←  音声  →|← tail_ms →|
フレーム開始      音声開始   音声終了   フレーム終了
```

- `head_ms`: フレームが始まってから音声が鳴り出すまでの時間
- `tail_ms`: 音声が終わってから次の文に移るまでの時間

<!--
各文の音声の前後に余白を設けることで、聞きやすいテンポになります。
-->

---

## threshold: 音声が長すぎる場合の処理

音声が割り当てフレームに収まらない場合、自動で速度を上げます。

速度の倍率が `threshold` を超えた場合は、
速度を上げる代わりにフレームを追加して倍率を下げます。

フレームが追加されると、その文の最後の字幕が繰り返し表示されます。

<!--
音声が長すぎる場合は自動でフレームを追加して速めすぎを防ぎます。
-->

---

## max_chars / break_ms

`max_chars`: 字幕 1 行の最大文字数。超えると次の文節から新しい字幕になります。

`break_ms`: 音声内の文節と文節の間に入れる間（ms）。
短くすると早口になり、長くするとゆったりした読み上げになります。

<!--
max_chars で字幕の折り返しを、break_ms で文節の間の長さを調整できます。
-->
