## markdown-cast: どうつかうの？

このツールは 4 つのステップで動画ができます。

1. 作業ディレクトリと markdown-cast を用意する
2. `deck.md` にスライドと発話ノートを書く
3. ビルドする
4. できた動画を再生する

## 費用を抑えながら進める 2 つの方法

1. **`limit`** — 最初の N ページだけ音声を作る
2. **`--dry-run`** — 音声合成を実行せずにパイプラインを確認する

この 2 つを組み合わせることで、費用を抑えながら作業を進められます。

1. **全角空白** — 発話ノートに「　」を入れる
2. **複合語辞書** — `mecab-private.dict.ss` に登録する

## 全角空白で区切り位置を指定する

発話ノートに全角空白「　」を入れると、その位置で文節を区切れます。

```
（例）
micro:bit を　使って　みよう
         ^^   ^^^^ ここで区切れる
```

## 複合語を 1 語として扱う

`micro:bit` のような語が複数の語に分割されるときは、
`mecab-private.dict.ss` に登録すると 1 語として扱えます。

```lisp

```

<!--

## 調整2: 句読点

句読点（。、）で文が自動的に区切られます。
1 つの文が音声の 1 つの読み上げ単位になります。

`build.ninja` の `strip_opt` で字幕・音声に含めるかどうかを設定します。

```
strip_opt = --strip-punctuation   # 句読点を落とす（既定）
strip_opt =                       # 句読点を残す
```

句読点を残すと、Azure が読み上げ時にブレス（息継ぎ）を入れます。

<!--
句読点で文が区切られ、1 つの文が音声の 1 つの読み上げ単位になります。
strip_opt で句読点を字幕と音声に含めるかどうかを設定します。
句読点を残すと Azure がブレスを入れ、自然な読み上げになります。
-->

## 調整3: 読み

`pronunciation.dict.ss` で固有名詞などの読み方を指定できます。


読みの変更は音声にだけ反映されます。字幕は元の表記のまま表示されます。

## 調整4: build.ninja のパラメータ

| パラメータ | 説明 | 既定値 |
|---|---|---|
| `capms` | 1 フレームの表示時間（ms） | 800 |
| `head_ms` | 各文の音声前の余白（ms） | 50 |
| `tail_ms` | 各文の音声後の余白（ms） | 10 |
| `threshold` | 速度調整の上限係数 | 1.3 |
| `max_chars` | 字幕 1 行の最大文字数 | 20 |
| `break_ms` | 文節間の間（ms） | 120 |

<!--
build.ninja のパラメータで動画の細かい挙動を調整できます。
-->

---

## capms: フレームの長さ

`capms` は 1 フレーム（字幕 1 枚）の表示時間です。

字幕の表示時間と動画のフレームレート（`1000/capms` fps）を両方決めます。

既定は 800ms（1 秒に 1.25 フレーム）。

<!--
capms は 1 フレームの表示時間です。
字幕の表示時間と動画のフレームレートを両方決めます。
-->

---

## head_ms / tail_ms: 各文の余白

各文の音声の前後に余白を設けます。

```
|←    head_ms    →|←  音声  →|← tail_ms →|
フレーム開始      音声開始   音声終了   フレーム終了
```

- `head_ms`: フレームが始まってから音声が鳴り出すまでの時間
- `tail_ms`: 音声が終わってから次の文に移るまでの時間

<!--
head_ms はフレームが始まってから音声が鳴り出すまでの余白です。
tail_ms は音声が終わってから次の文に移るまでの余白です。
-->

---

## threshold: 音声が長すぎる場合の処理

音声が割り当てフレームに収まらない場合、自動で速度を上げます。

速度の倍率が `threshold` を超えた場合は、
速度を上げる代わりにフレームを追加して倍率を下げます。

フレームが追加されると、その文の最後の字幕が繰り返し表示されます。

<!--
threshold は音声を速めるかフレームを追加するかの境界です。
倍率が threshold を超えるとフレームを追加して速めすぎを防ぎます。
フレームが追加されると最後の字幕が繰り返し表示されます。
-->

---

## max_chars / break_ms

`max_chars`: 字幕 1 行の最大文字数。超えると次の文節から新しい字幕になります。

`break_ms`: 音声内の文節と文節の間に入れる間（ms）。
短くすると早口になり、長くするとゆったりした読み上げになります。

--pitch・--rate・--style オプションが tts2wav.ros に実装されています（88〜104 行）。ただし専用の ninja 変数はなく、key.ninja の tts_opt 経由で渡します。

tts_opt = --pitch -10% --rate 5%

- pitch: 相対値（-10% で低く、+10% で高く）
- rate: 相対値（-15% でゆっくり、+15% で速く）
- style: Azure の話し方スタイル（cheerful、calm など）
