# markdown-cast

Marp のスライドから、字幕・音声つき動画を作るパイプライン。

---

## 何ができるか

`deck.md` ひとつから、こうなる。

| 出力 | 内容 |
|---|---|
| `deck.pdf` | Marp の PDF |
| `subs.srt` | 字幕ファイル |
| `*-with-caption.mp4` | 字幕焼き込み動画（音声なし） |
| `*-final-caption-audio.mp4` | 字幕＋ナレーション音声つき動画 |

<!--
markdown-cast は、Marp の Markdown スライドを起点に、字幕と音声つきの動画を作るパイプラインです。
入力は deck.md ひとつだけ。
字幕ファイル、字幕を焼き込んだ動画、最終的には音声ナレーションまでついた動画が出てきます。
-->

---

## なぜこれが必要か

音声・字幕・画像を別々に作ると、**タイミングがずれる**。

- 音声が長すぎて字幕が追いつかない
- スライドが変わるタイミングと話が合わない
- 手作業で合わせるのは現実的でない

markdown-cast はこれを**自動で解決**する。

<!--
動画制作でいちばん厄介なのが、音声・字幕・画像の同期です。
音声が長すぎて字幕が追いつかない、スライドが変わるタイミングと話がずれる、こういったことが起きます。
手作業で合わせるのは現実的でないので、markdown-cast はこれを自動で解決する仕組みを持っています。
-->

---

## 中心の考え方

**文節 = 1フレーム = 一定時間（既定 800ms）**

- 形態素解析で文章を「文節」に分割する
- 文節ひとつが動画の 1フレームに対応する
- 字幕・音声・画像すべてが、この同じ番号体系で動く

どれかひとつでもずれることがない。

<!--
このパイプラインの核心は、文節という単位を共通の時間軸にすることです。
形態素解析で文章を文節に分割し、文節ひとつを動画の 1フレームに対応させます。
字幕も、音声を差し込む位置も、スライド画像も、すべてこの同じ番号で管理します。
これによって、どれかひとつがずれるということがなくなります。
-->

---

## 入力の書き方

Marp の HTML コメントに**発話ノート**を書く。

```markdown
## スライドのタイトル

スライドの本文。

<!--
読み上げ原稿をここに書く。
句点で文が切れ、それが TTS の読み上げ単位になる。
全角スペース「　」で文節を手動で区切ることもできる。
-->
```

スライドの見た目と原稿が、1ファイルに同居する。

<!--
入力ファイルの書き方は、Marp のスライドに HTML コメントで発話ノートを追加するだけです。
コメントの中に読み上げ原稿を書きます。
句点「。」のところで文が切れて、それが TTS の読み上げ単位になります。
全角スペースを使えば、文節の区切りを手動で指定することもできます。
スライドの見た目と原稿が 1ファイルに収まるので、ズレが起きません。
-->

---

## パイプラインの流れ

```
deck.md
 ├─ 発話ノートを取り出す（note2ss）
 │   └─ 形態素解析・文節化（mecab）
 │       ├─ 字幕を作る: SRT
 │       │   └─ スライド PNG に字幕を焼き込む → 動画
 │       └─ 読み上げ単位を作る
 │           └─ Azure TTS で wav を生成 ★課金
 │               └─ タイミング計算・速度調整・音声を重ねる
 └─ 通し番号を振り直して最終版を作る
```

<!--
処理の流れはこのようになっています。
まず deck.md から発話ノートを取り出し、形態素解析で文節に分解します。
ここから字幕の枝と音声の枝に分かれます。
字幕はスライド PNG に焼き込まれて動画になります。
音声は Azure TTS で wav を生成し、タイミングを計算して動画に重ねます。
最後に、速度調整でフレームを足した分の通し番号を振り直して、最終版を作ります。
-->

---

## 実行コマンド

`build.ninja` を用意して `ninja` を叩くだけ。

```sh
ninja srt                     # 字幕だけ作る
ninja with-caption.mp4        # 字幕焼き込み動画（Azure 不要）
ninja wav                     # ★ Azure TTS で音声生成（課金）
ninja final-caption-audio.mp4 # 最終動画
```

`ninja wav` **以外は Azure 不要**。まず音声なしで試せる。

<!--
実行は、build.ninja を用意して ninja コマンドを叩くだけです。
字幕だけ作る、字幕焼き込み動画を作る、音声を生成する、最終動画を作る、という段階を選べます。
ninja wav 以外は Azure を使わないので、まず音声なしで字幕と動画の確認ができます。
-->

## build.ninja のパラメータ

| パラメータ | 既定値 | 意味 |
|---|---|---|
| `capms` | 800 | 1フレームの長さ（ms） |
| `max_chars` | 10 | 1字幕の最大文字数 |
| `threshold` | 1.3 | この倍速を超えるとフレームを追加 |
| `head_ms` | 50 | 各文の頭の余白（ms） |
| `voice` | — | Azure TTS の音声名 |

`capms` だけ全ステップで一致させれば、あとは好みで調整できる。

<!--
build.ninja の冒頭にパラメータをまとめてあります。
capms が 1フレームの長さで、全ステップで一致させる必要があります。
max_chars で字幕の折り返し文字数、threshold で速度調整の上限を決めます。
Azure TTS の音声名も、ここで指定します。
capms さえ合っていれば、あとは好みで調整してください。
-->

## まとめ

- `deck.md` に発話ノートを書くだけで動画になる
- 字幕・音声・画像は**文節番号**で自動同期される
- Azure なしでも字幕つき動画まで作れる
- パラメータは `build.ninja` の冒頭で一元管理

まず `ninja with-caption.mp4` から試してみてください。
